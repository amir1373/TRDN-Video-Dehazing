  - `TRAIN_ROOT=/content/drive/MyDrive/REVIDE_sequences/Train`
  - `TEST_ROOT=/content/drive/MyDrive/REVIDE_sequences/Test`

  - `FLOW_TRAIN=/content/drive/MyDrive/video_dehaze_flows/train`
  - `FLOW_VAL=/content/drive/MyDrive/video_dehaze_flows/val`

- Ensured REVIDE remains the only training source for diffusion, temporal learning, flow guidance, and reference retrieval.
- Added shape assertions for frames, masks, latents, flows, warped references, temporal memory, and reference weights.
- Added `TemporalRetrievalTransformer` as an integrated temporal reasoning block.
- Updated reference selection to accept transformer prior logits.
- Added PEFT LoRA adapter support through `ENABLE_LORA`/`enable_lora`.
- Added optional `torch.compile`, xFormers, gradient checkpointing, and mixed precision hooks.
- Added debug forward script that saves intermediate outputs.
- Added dataset inspector notebook.
- Updated scripts to use centralized config defaults rather than requiring dataset paths.

## Shape Issues Fixed

The following shape contracts are now asserted:

```text
frames:            [B,T,3,H,W]
current_frame:     [B,3,H,W]
target_frame:      [B,3,H,W]
mask:              [B,1,H,W]
warped_references: [B,T-1,3,H,W]
flows:             [B,T-1,2,H,W]
temporal_memory:   [B,C,H,W]
reference_weights: [B,T-1,H,W]
latents:           [B,4,H/8,W/8]
```

## Training Issues Fixed

- Training now routes memory through ConvLSTM and the Temporal Retrieval Transformer before reference selection.
- Reference selection now combines learned per-pixel logits with transformer prior logits.
- Training uses the same REVIDE sequence samples for both diffusion and temporal objectives.
- `reconstruct` mode corrupts the current clean frame synthetically and keeps clean REVIDE as the target.
- `dehaze` mode uses hazy input sequence and clean current target.
- Gradient norm clipping remains enabled; gradient norm logging is recommended as a next telemetry addition.

## Memory Issues Found

- Full Stable Diffusion inpainting UNet plus RAFT at 256x256 is still heavy. Default batch size remains `1`.
- Local validation ran on CPU, so CUDA memory usage could not be measured here.
- `debug_forward_pass.py` records CUDA allocated/reserved/max memory when run on Colab GPU.

## Remaining Risks

- Real REVIDE folder naming may still vary. Run `notebooks/DATASET_INSPECTOR.ipynb` in Colab before training.
- RAFT flow direction should be visually inspected per dataset with `debug_forward_pass.py` outputs.
- Stable Diffusion checkpoint downloads may require Hugging Face access depending on runtime policy.
- LoRA adapter behavior depends on compatible `diffusers` and `peft` versions.

## Recommendations

- First Colab run:
  1. Open `notebooks/DATASET_INSPECTOR.ipynb`.
  2. Verify sequence counts and image sizes.
  3. Run `python debug_forward_pass.py --no-raft --skip-diffusion`.
  4. Run `python debug_forward_pass.py` on a GPU runtime to verify RAFT and diffusion outputs.
- Start training with `TRAIN_MODE="reconstruct"` and `IMAGE_SIZE=256`.
- Keep `BATCH_SIZE=1`, mixed precision, xFormers, and gradient checkpointing enabled.
- Enable LoRA after the full-UNet baseline path is verified.
